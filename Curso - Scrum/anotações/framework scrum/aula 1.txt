Aula 1
============================================================================

O Scrum é um framework de execução e planejamento de entrega de serviços ou
ou produtos de forma interativa e incremental, ou seja, é feito pequenas entregas
no intervalo de 2~4 semanas, que são chamadas de *sprint*.
O Scrum é dividido em 8 passos, o maindset ágil.


Como era antes do Scrum
---
Water fall -> conceito onde todo projeto é desenvolvido por completo e entregue
no final.
_____________

Pilares do Scrum
---

1 - Transparência

2 - Inspeção

3 - Adaptação

Essa é a TIA do Scrum, a TIA deve estar sempre presente no mindset ágil.
_____________

Processos do Scrum
---
Product Backlog:
[
  Momento do processo onde o PO entra em contato com o stakeholder e geração
  uma PBI.
]
Sprint planning:
[
  Uma cerimônia que acontece com o objetivo de mostrar o PBI para o time de desenvolvimento.
]
Sprint backlog:
[
  Uma fatia do PBI que é entregue ao time de desenvolvimento.
]
Daily Scrum:
[
  Reuniões diarias, com intuito de alinhas o processo de desenvolvimento de todos os
  membros do time de dev, ela é organizada para ser feita após um dia de trabalho.
]
Sprint Reviw:
[
  Após 2~4 semanas é feito uam revisão para saber se está tudo ok.
]
Increment:
[
  Parte do processo onde aplicações/funcionalidades são incrementadas.
]
Sprint retrospective:
[
  É feito uma retrospectiva, e logo após isso, o processo se repete a partir do ponto da
  sprint planning.
]
_____________


Entregas fatiadas -> para o processo ágil ser efetivo, toda entrega deve agregar valor,
a entrega deve trazer funcionalidades, ou seja, não é entregue um produto que não tenha
funcionalidade, uma tela somente com o design ou somente o código usado na API. O real
objetivo é entregar um item que ele já possa utilizar, não somente a interface gráfica
ou a APi, mas a interface gráfica integrada a API.


============================================================================
Dicionário:
Product owner -> responsável do projeto, representa os stakeholders.
stakeholder -> cliente interessado no desenvolvimento do produto/negócio.
PBI -> product backlog itens
PBI -> Uma lista de tarefas que precisam ser desenvolvidas para construir o produto.
